# Vehicle Care OS (VCO) First Five Core Screens

## Purpose

This document defines the first five core screens to design for `VCO`, using the `Imowash` reference as directional inspiration while keeping the product grounded in the MVP requirements.

These screens are the highest-leverage starting point because they define the customer booking loop and the operational workflow behind it.

## Screen 1: Homepage Hero / Landing Entry

### Goal
Communicate premium vehicle care operations software in under 10 seconds.

### Primary User
- operator prospect
- business owner
- manager

### Key Message
`The Operating System For Modern Vehicle Care`

### Must-Have Elements
- strong headline
- subheadline explaining bookings, memberships, technicians, payments, and customer experience
- primary CTA
- secondary CTA
- product or workflow visual
- trust strip or proof statement

### Layout Direction
- bold headline with aggressive whitespace
- black, off-white, and blue palette
- product frame or abstract system visualization instead of generic hero stock image
- immediate visual sense of premium control

### Reference Influence From Imowash
Borrow:
- bold minimal composition
- premium, high-contrast presentation
- strong type-led visual hierarchy

Do not copy:
- exact typography treatment
- exact composition or layout structure

## Screen 2: Customer Booking Flow

### Goal
Allow a customer to book a service quickly and confidently.

### Primary User
- customer

### Core Steps
- select vehicle
- select service package
- choose add-ons
- choose location or mobile service
- select time slot
- review and pay

### Must-Have Elements
- visible progress stepper
- concise package information
- clear pricing summary
- selected vehicle summary
- time-slot selection UI
- review state before payment

### Layout Direction
- linear and mobile-friendly
- strong hierarchy on step title and selected data
- minimal distractions
- pricing and confirmation clarity prioritized over decoration

### Reference Influence From Imowash
Borrow:
- clean app-first feeling
- premium simplicity
- controlled use of accent blue for actions and selected states

Do not copy:
- exact card design
- exact subscription panel styling

## Screen 3: Operator Dashboard

### Goal
Give operators a fast understanding of the business state.

### Primary User
- operator
- manager

### Must-Have Elements
- KPI row for bookings, revenue, memberships, completed jobs
- today’s queue summary
- exceptions or alerts panel
- location filter
- quick links into bookings and team workflows

### Layout Direction
- dense but highly ordered
- information hierarchy led by large numeric moments
- cards only where they structure meaning
- dashboard should feel operational, not decorative

### Reference Influence From Imowash
Borrow:
- sharp, minimal presentation
- strong numeric emphasis
- clear state visibility

Expand beyond reference:
- more business depth
- more filters and operational context

## Screen 4: Operator Booking Detail

### Goal
Let an operator manage a booking from one screen.

### Primary User
- operator

### Must-Have Elements
- booking status and timeline
- customer summary
- vehicle summary
- selected service package and add-ons
- payment state
- technician assignment
- notes section
- actions for reschedule, cancel, reassign, update status

### Layout Direction
- split layout with details on left and actions on right, or stacked mobile equivalent
- clear separation between service facts and operator actions
- state and assignment should be visible at the top of the screen

### Reference Influence From Imowash
Borrow:
- clean systemized interface feel
- premium treatment of service information

Expand beyond reference:
- this is a true operating screen, so it should be more function-dense than the customer-facing inspiration

## Screen 5: Technician Job Detail

### Goal
Help the technician execute the job with minimal friction.

### Primary User
- technician

### Must-Have Elements
- vehicle info
- service package and add-ons
- checklist
- large status actions
- notes input
- photo upload for before and after states
- completion control

### Layout Direction
- mobile-first
- large tap targets
- minimal text density
- task-first flow with status actions pinned near bottom if mobile

### Reference Influence From Imowash
Borrow:
- app-first confidence
- simple service-state communication

Expand beyond reference:
- more operational utility for actual field or bay work

## Why These Five Screens First

These five screens establish:
- market-facing positioning
- customer conversion path
- operational command center
- booking management depth
- service execution workflow

Without these screens, the product still lacks its core loop.

## Recommended Design Order

Design in this sequence:
1. homepage hero
2. customer booking flow
3. operator dashboard
4. operator booking detail
5. technician job detail

## Recommended Deliverables Per Screen

For each screen, create:
- low-fidelity wireframe
- content hierarchy
- UI state notes
- mobile and desktop behavior
- component inventory

## Follow-Up Screens After These Five

Next screens should be:
- customer home
- membership overview
- service history detail
- customers CRM detail
- services management
